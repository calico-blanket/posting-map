# 今後の開発ロードマップ

このドキュメントでは、プロジェクトの次のフェーズにおける主要な開発目標と要件を定義します。

## フェーズ 1: セキュリティと品質の担保 (要件5)

コードベースの品質を維持し、ユーザーデータを保護するための自動化とセキュリティ対策を強化します。

### 目標
- 脆弱性の早期発見と修正。
- コード品質の統一。
- データの安全な保護。

### 実装計画
1.  **CI/CDパイプラインの構築 (GitHub Actions)**
    - **Lint & Type Check**: プルリクエスト作成時に `ESLint` と `TypeScript` のチェックを自動実行。
    - **Security Scan**: `CodeQL` 等のセキュリティスキャンツールを導入し、脆弱性を定期チェック。
2.  **Firebaseセキュリティルールの強化**
    - 現在のルールをレビューし、本番運用に耐えうる詳細なアクセス制御（Allowlist等）を実装。
    - 位置情報データの読み書き権限を厳格化。

## フェーズ 2: データ管理機能の拡張 (要件6)

ユーザーが自身のデータを包括的に管理・活用できるように、エクスポート・インポート機能を強化します。

### 目標
- 配布エリアだけでなく、登録したスポット情報（写真、建物名、緯度経度、メモ）を外部データとして活用可能にする。
- データのバックアップと復元を容易にする。

### 実装計画
1.  **CSV/スプレッドシート エクスポート**
    - 登録済みのスポットデータをCSV形式でダウンロード可能にする。
    - **出力項目**: 画像URL, 建物名, 緯度, 経度, メモ, 登録日時, 作成者ID。
2.  **データインポート（復元）**
    - CSVデータを読み込み、アプリ上のデータを復元・更新する機能。
    - 画像URLのリンク切れハンドリング等を考慮。

## フェーズ 3: 非エンジニア向け導入の簡略化 (要件4)

専門知識がないユーザーでも、このアプリケーションを簡単に利用・導入できるようにします。

### 目標
- 複雑な環境設定（Vercel/Firebaseコンソール操作）の排除または自動化。
- アプリのような手軽なインストール体験。

### 提案・検討案
1.  **PWA (Progressive Web App) の最適化**
    - 現状のPWA機能を強化し、ストアアプリと同等のUXを提供（オフライン動作の安定化、プッシュ通知等）。
    - *メリット*: 最も低コストで全プラットフォームに対応可能。
2.  **デスクトップアプリ化 (Electron / Tauri)**
    - PCユーザー向けに、インストーラー形式で配布可能なネイティブアプリを作成。
    - *メリット*: ローカルファイルシステムへのアクセスが容易、ブラウザの制約を受けにくい。
3.  **簡易デプロイメントキット**
    - "Deploy to Vercel" ボタンのような、1クリックで環境構築が完了するテンプレートの整備。
    - 必要なAPIキー設定をガイダンスするセットアップウィザードの実装。
